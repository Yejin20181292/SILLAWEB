CSS Responsive Tabs

Responsive & abstarct

v.2

Price: 2,500$

based on hours spent

special thanks to mikestreety for his idea!

Characteristics:

  1. Variable Tab Content Heights
  2. General rules (not per tab selectors as in other solutions)
  3. Tab content animation
  4. Huge number of tabs leads to wrapped behaviour
  5. For screens below 450px it converts to vertical stacked tab labels
  6. It can be converted to vertical accordion
  7. Depth of tags only 1
  8. core code 58 lines uncompressed(!) + extra presentational classes which you can delete(!)

HTML structure:

  1. level 0: <div class="tabs"> or <div class="tabs shaded op-70 tabs-outline">
  2. level 1: <input type="radio">
  3. level 1: <label>
  4. level 1: <div class="tab">
  5. repeat steps 2-4

CSS structure:

  1. in file you can delete blocks of code connected with presentational classes you don't need

Presentational Classes (at root):

  1. "tabs-top-line": sets a line on top of active tab label
  2. "shaded": sets background colors
  3. "tabs-outline": sets a line around tab labels
  4. "content-outline": sets a line around the content
  5. "tabs-align-left": aligns tab labels to the left (default is justified)
  6. "tabs-align-right": aligns tab labels to the right
  7. "tabs-align-center": aligns tab labels to the center consequently
  8. "tabs-align-center-solo": aligns tab labels to the center seperated
  9. "tabs-rtl": reverses tab label order (first label starts at the right side)
  10. "op-70": in combination with "shaded" sets background opacity to 0.7 (change or add rules for different opacities)
  11. "slide-left": content slides to left
  12. "slide-right": content slides to right
  13. "slide-top": content slides to top
  14. "slide-bottom": content slides to bottom
  15. "material": a material theme
  16. "accordeon-vertical": a vertical accordeon
  17. any combination

Behavioral Classes (programming):

  1. apply property "checked" at a hidden input

Screening Requirements:

  1. should allow tab label stacking
  2. should contain css rules as a generic solution for any number of tabs

End Result

Flex box satisfies all the requirements. No js is needed. By default the tabs are transparent allowing impressive designs with containers that have background images. Another default behavior is the distribution of tab labels which ocupy all the available space (justified).

If we need scrolling tabs then a solution following Bootstrap's tag organization is better with a bit of javascript; the tab labels are organized in one group and the contents into another. Even this solution can be transformed to a such organization as long as we replace option's checked rules with a simple class like ".active" and replace flex box approach with floats. On the other hand, floats are notorious for bad designs when they start to wrap - clearfix hack doesn't solve the problem.

Bug: just before the media query break point where wrapping happens, we can see all tab alignements except the default (justified) and "tabs-align-center-solo" to wrap to uneven positions which is logical as these alignments have no vertical axis symmetry.
